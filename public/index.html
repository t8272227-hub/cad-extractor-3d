🖱️ Навигация на холсте

Мышь

  • Колесо ↕ — масштабирование

Мобильные / Touch

  • Один палец — панорамирование
  • Pinch-Zoom — масштабирование двумя пальцами

💡 Привязка к узлам DXF

При активном инструменте «Отметить точку» курсор автоматически притягивается к ближайшему узлу чертежа. Жёлтый кружок — «свободный» клик, зелёный кружок — привязка к узлу.

📐 Режим Чертёж (DXF)

Открытие файла

Нажмите кнопку «Открыть» (шапка) и выберите ASCII DXF файл.

⚠️ Бинарный формат DWG/DXF не поддерживается. В AutoCAD: Сохранить как → R12/LT2 DXF (ASCII).

Инструменты левой панели

Отметить точку

ЛКМ по узлу чертежа — создаёт геодезическую точку с координатами X,Y. Привязка к узлам автоматическая.

Добавить точку с интерполяцией Z

Добавляет точку и сразу вычисляет высоту Z методом IDW по соседним точкам с известной высотой.

Размерная линия

Два клика — первая и вторая точки. Shift фиксирует угол (0°/45°/90°). Результат в метрах в таблице «Размеры».

Область для PDF

Выделите прямоугольную область чертежа, которая попадёт в PDF-отчёт. Синяя рамка показывает выделение.

Горизонтали (топо-линии)

Строит изолинии высот по триангуляции Делоне. Задайте шаг в метрах. Требует минимум 3 точки с Z.

Параметры слоёв

Панель управления слоями DXF. Можно скрывать слои, добавлять описания, выбирать слои для включения в PDF. Панель перетаскивается за заголовок.

Подписи точек

Включает/выключает отображение номеров и Z-высот точек прямо на холсте.

Цвет линий

Меняет цвет отображения линий чертежа. Маленький кружок на кнопке показывает текущий цвет.

Задать Z-высоту (правая панель)

Выберите точку из списка → введите высоту → выберите тип: Абс. (ортометрическая высота в метрах) или Отн. (относительная от условного нуля).

«Интерполировать все» — автоматически вычисляет Z для точек без высоты методом IDW (обратных взвешенных расстояний).

✏️ Режим Ручной ввод

Инструменты рисования

  • Панорамирование — перемещение по холсту (ЛКМ + перетащить)
  • Линии — ЛКМ начало, ЛКМ конец. Shift = орто-режим (0/90°). ПКМ = отменить
  • Полигон — последовательные клики. Закрыть кнопкой «Завершить» или ПКМ

Добавить точку вручную

Введите X, Y (и Z опционально) в левой панели → «Добавить».

Таблицы «Точки» и «Линии» позволяют редактировать и удалять элементы.

📥 Импорт координат

TXT / CSV — файл с разделителями (пробел, табуляция, запятая или точка с запятой). Поддерживаемые форматы строк:

X Y ← две колонки
X Y Z ← три колонки
N X Y Z ← номер + координаты
N X Y Z TYPE ← с типом точки

SDR (Sokkia) — файл сырых данных тахеометра. Автоматически распознаётся по заголовку формата Sokkia SDR33.

⚠️ Разделитель десятичных: точка. Кириллица в именах слоёв — windows-1251.

📤 Экспорт

DXF — все точки как POINT-объекты, линии как LINE/POLYLINE
CSV — таблица координат в текстовом формате (N;X;Y;Z)
PDF — ГОСТ-вёрстка: чертёж (слева) + штамп и таблица (справа)

🖼️ Фоновое изображение

Загрузите растровую подложку (PNG/JPG) через кнопку в панели. Настраивается: сдвиг, масштаб, прозрачность. Удобно для оцифровки бумажных планов.

🎯 Геопривязка чертежа

Что это такое?

Геопривязка позволяет наложить съёмочный DXF (в условной системе координат) поверх кадастрового плана (в геодезической системе). Используется, когда нужно сопоставить результаты полевой съёмки с кадастровыми данными.

Пошаговая инструкция

  1. 1 Откройте кадастровый DXF (основной файл с абсолютными геодезическими координатами). Убедитесь, что чертёж виден на экране.
  2. 2 Нажмите «Геопривязка» в шапке. Откроется модальное окно.
  3. 3 Шаг 1 — Введите локальные координаты двух опорных точек из съёмочного файла (например, 0;0 и 30.5;0).
  4. 4 Шаг 2 — Укажите геодезические координаты тех же точек. Используйте кнопку «На карте» — кликните по нужной точке прямо на кадастровом чертеже, координаты заполнятся автоматически.
  5. 5 Шаг 3 — Загрузите съёмочный DXF.
  6. 6 «Применить и совместить» — вычисляет трансформацию и накладывает съёмочный файл оранжевым поверх кадастрового.

Математика: 2D Helmert Similarity Transform

По двум парам точек вычисляются 4 параметра:

s = масштаб = |P2_geo − P1_geo| / |P2_local − P1_local|
θ = угол поворота = atan2(P2_geo) − atan2(P2_local)
tx, ty = сдвиг = P1_geo − s·R(θ)·P1_local

Невязка (residual) в блоке информации = ошибка подгонки во второй контрольной точке. В идеале < 0.001 м.

Зеркальное отображение (применяется перед геопривязкой)

При полевой съёмке координаты могут быть записаны в зеркально отражённой системе относительно кадастра — в зависимости от ориентации базовых точек прибора и направления оцифровки.

💡 Если необходимо — можно включить оба зеркала одновременно. Зеркалирование применяется в локальной системе координат до трансформации Helmert.

📊 Инструменты анализа

Интерполяция Z (IDW)

Метод обратных взвешенных расстояний. Для каждой точки без высоты вычисляется Z на основе расстояний до точек с известными высотами. Чем ближе точка — тем больше её влияние.

Требования: минимум 3 точки с известным Z.

Горизонтали (топо-линии)

Строит топографические изолинии по триангуляции Делоне. Задайте шаг (м). Линии сглаживаются. Итоговые горизонтали можно скрыть/показать чекбоксом.

Требования: минимум 4 точки с Z.

3D Визуализация (TIN)

Строит 3D-поверхность рельефа (TIN — Triangulated Irregular Network) по точкам с Z. Градиентная раскраска по высоте. Управление: ЛКМ — вращение, ПКМ — сдвиг, колесо — зум.

Wireframe-режим — переключатель каркасной сетки.

Объёмы земляных работ

Вычисляет объёмы выемки и насыпи относительно заданной плановой отметки (уровень разбивки). Площадь считается по выпуклой оболочке (Convex Hull) всех точек или по нарисованному контуру.

Вывод: объём выемки (м³), объём насыпи (м³), превышение.

Кадастровый поиск (Ручной режим)

Поиск границ земельного участка по кадастровому номеру через API Росреестра (pkk.rosreestr.ru) или OSM. Загруженные границы конвертируются из WGS-84 (lon/lat) в метрическую проекцию и отображаются на холсте.

Ориентация: A4 альбомная. Поля: 8 мм. После нажатия «Применить» откроется диалог печати браузера — выберите «Сохранить как PDF».

Настройки PDF (модальное окно)

  • Название проекта — выводится как заголовок в штампе
  • Исполнитель — ФИО или организация
  • Логотип — PNG/JPG, максимум ~55px высотой в штампе
  • Слои в PDF — слои с включённым чекбоксом 📄 в панели слоёв выводятся в таблицу

📁 Поддерживаемые форматы

Входные форматы

  • .dxf — ASCII DXF (R12 / R14 / R2000+). Поддерживаемые примитивы: LINE, LWPOLYLINE, POLYLINE, SPLINE, CIRCLE, ARC, POINT, TEXT, MTEXT, INSERT (блоки). DXF-слои распознаются автоматически.
  • .txt .csv — таблица координат. Разделители: пробел, табуляция, запятая, точка с запятой. Колонки: [N] X Y [Z] [Type].
  • .sdr — Sokkia SDR33. Файл сырых данных тахеометра. Парсируются записи типа 08 (координаты точек).
  • .png .jpg — растровая подложка (только ручной режим).

Выходные форматы

  • .dxf — экспорт из ручного режима. Точки как POINT, линии как LINE. Кодировка windows-1251.
  • .csv — таблица координат: N;X;Y;Z. UTF-8.
  • .pdf — отчёт через браузерный print. A4 альбом, ГОСТ-вёрстка.
